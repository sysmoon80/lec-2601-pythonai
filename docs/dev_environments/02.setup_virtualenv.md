## 1. 가상환경이란 무엇인가?

### 1.1 가상환경의 필요성

**가상환경(Virtual Environment)**은 프로젝트별로 독립적인 Python 실행 환경을 제공합니다:

- **의존성 격리**: 프로젝트마다 다른 버전의 패키지 사용 가능
- **충돌 방지**: 시스템 Python과 분리되어 영향 없음
- **깔끔한 관리**: 불필요한 패키지로 시스템 오염 방지

### 1.2 UV 가상환경 vs 기존 방식

| 구분 | 기존 방식 (venv) | UV 방식 |
|------|------------------|---------|
| 생성 명령 | `python -m venv .venv` | `uv venv` |
| 생성 속도 | 느림 | 매우 빠름 |
| 패키지 관리 | `pip install` | `uv add` |
| 의존성 관리 | `requirements.txt` | `pyproject.toml` + `uv.lock` |

## 2. 가상환경 생성

### 🎯 Step 1: 프로젝트 폴더 이동

```bash
# 프로젝트 폴더로 이동
cd lec-2601-pythonai
```

### 🎯 Step 2: 가상환경 생성

```bash
# 가상환경 생성 (.venv 폴더 생성)
uv venv
```

**예상 출력**:
```
Using Python 3.12.1 interpreter at: /Users/username/.local/share/uv/python/cpython-3.12.1/bin/python3.12
Creating virtualenv at: .venv
Activate with: source .venv/bin/activate
```

### ✅ 확인: 가상환경 폴더 확인

```bash
# macOS/Linux
ls -la .venv/

# Windows
dir .venv
```

**예상 결과**: `bin`, `lib`, `include` 폴더가 생성되어야 합니다.

## 3. 가상환경 활성화/비활성화

### 🎯 Step 3: 가상환경 활성화

#### macOS/Linux

```bash
# 가상환경 활성화
source .venv/bin/activate
```

#### Windows (CMD)

```cmd
.venv\Scripts\activate
```

### ✅ 확인: 활성화 확인

활성화되면 터미널 프롬프트에 가상환경 이름이 표시됩니다:

```bash
(.venv) username@hostname project-folder %
```

zsh 환경에서는 다른 형식으로 보여집니다.

```zsh
 ~/h/l/lec-2601-pythonai │ main ?6                   ✔ │ lec-2601-pythonai  │ 21:23:49 
```

### 🎯 Step 4: 가상환경 비활성화

```bash
# 가상환경 비활성화
deactivate
```

## 4. 패키지 설치 및 관리

UV는 프로젝트 기반 패키지 관리를 지원합니다. `pyproject.toml` 파일이 있다면 `uv add` 명령어를 사용합니다.

### 🎯 Step 5: 단일 패키지 설치

```bash
# requests 패키지 추가
uv add requests
```

**예상 출력**:
```
Resolved 1 package in 0.12s
Added 1 package in 0.05s
 + requests==2.32.3
```

> **참고**: `uv add`는 `pyproject.toml`에 의존성을 자동으로 기록하고 `uv.lock` 파일을 생성/업데이트합니다.

### 🎯 Step 6: 여러 패키지 한 번에 설치

```bash
# 여러 패키지 동시 추가
uv add requests numpy pandas
```

### ✅ 확인: 설치된 패키지 목록

```bash
# 설치된 패키지 목록
uv pip list
```

**예상 출력**:
```
Package    Version
---------- -------
numpy      2.1.3
pandas     2.2.3
requests   2.32.3
```

### 🎯 Step 7: 패키지 삭제

```bash
# 패키지 제거
uv remove requests
```

## 5. 의존성 파일 관리

UV는 `pyproject.toml`과 `uv.lock` 파일을 사용하여 의존성을 관리합니다.

### 🎯 Step 8: pyproject.toml 확인

`uv add`로 패키지를 추가하면 `pyproject.toml`이 자동으로 업데이트됩니다.

```bash
# pyproject.toml 내용 확인
cat pyproject.toml
```

**pyproject.toml 예시**:
```toml
[project]
name = "lec-2601-pythonai"
version = "0.1.0"
requires-python = ">=3.12"
dependencies = [
    "requests>=2.32.3",
    "numpy>=2.1.3",
    "pandas>=2.2.3",
]
```

### 🎯 Step 9: 다른 환경에서 동기화

다른 환경에서 동일한 패키지들을 설치할 때:

```bash
# 프로젝트 복제 후
git clone <repository-url>
cd lec-2601-pythonai

# 의존성 동기화 (pyproject.toml + uv.lock 기반)
uv sync
```

> **참고**: `uv sync`는 `pyproject.toml`과 `uv.lock`을 읽어 정확히 같은 버전의 패키지들을 설치합니다.

### 📌 선택사항: requirements.txt 호환

기존 `requirements.txt`를 사용해야 하는 경우:

```bash
# requirements.txt에서 패키지 설치
uv pip install -r requirements.txt

# 또는 requirements.txt를 pyproject.toml로 변환
uv add -r requirements.txt
```

## 6. 프로젝트 생성

UV는 가상환경 외에도 프로젝트 템플릿 생성 기능을 제공합니다.

### 🎯 Step 10: 새 프로젝트 생성

이미 lec-2601-python 프로젝트(https://github.com/sysmoon80/lec-2601-pythonai.git) 에서 uv 를 이용하여 프로젝트 환경을 구성했습니다.
따라서 아래 과정을 이미 수행하여 프로젝트 구조가 구성되어 있습니다.
신규 프로젝트 생성하여 별도 테스트 해보시기 바랍니다.

```bash
# 새 프로젝트 생성 (가상환경 + 기본 파일 자동 생성)
uv init my-new-project
```

**생성되는 파일 구조**:
```
my-new-project/
├── .python-version
├── .venv/
├── pyproject.toml
└── README.md
```

### ✅ 확인: 프로젝트 구조 확인

```bash
# 프로젝트 폴더로 이동
cd my-new-project

# 내용 확인
ls -la
cat pyproject.toml
```
