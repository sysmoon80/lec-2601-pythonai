## 1. Git이란 무엇인가?

### 1.1 Git 소개

**Git**은 분산 버전 관리 시스템(DVCS)으로, 소스 코드의 변경 사항을 추적하고 여러 개발자가 협업할 수 있게 해줍니다.

- **버전 관리**: 파일의 변경 이력을 기록하고 이전 버전으로 복구
- **협업 지원**: 여러 개발자가 동시에 작업 가능
- **브랜칭**: 독립적인 작업 공간에서 실험적인 기능 개발
- **분산 관리**: 각 개발자가 전체 이력을 로컬에 보유

### 1.2 Git vs GitHub

| 구분 | Git | GitHub |
|------|-----|--------|
| **정의** | 버전 관리 시스템 | Git을 사용하는 호스팅 서비스 |
| **역할** | 로컬에서 버전 관리 수행 | 원격 저장소 제공, 협업 지원 |
| **예시** | git commit, git branch | Repository, Pull Request, Issues |

### 1.3 Git 기본 용어

| 용어 | 설명 |
|------|------|
| **Repository (레포지토리)** | 프로젝트의 모든 파일과 이력을 저장하는 곳 |
| **Commit (커밋)** | 변경 사항을 확정하고 저장하는 작업 |
| **Branch (브랜치)** | 독립적인 작업 흐름 |
| **Clone (클론)** | 원격 레포지토리를 로컬로 복사 |
| **Push (푸시)** | 로컬 변경 사항을 원격에 업로드 |
| **Pull (풀)** | 원격 변경 사항을 로컬에 다운로드 |

## 2. macOS에 Git 설치

### 🎯 Step 1: Homebrew를 사용한 설치 (추천)

> **실습**: 터미널을 열고 Homebrew로 Git을 설치해보세요.

#### 1-1. Homebrew 설치 확인

```bash
# Homebrew가 이미 설치되어 있는지 확인
brew --version
```

**Homebrew가 설치되어 있지 않다면**:

```bash
# Homebrew 설치 스크립트 실행
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

**설치 완료 후 안내 사항**:
- 설치 완료 후 터미널에 표시되는 "Next steps"를 따름
- Apple Silicon(M1/M2/M3)의 경우 추가 설정 필요:
```bash
# Apple Silicon 전용 경로 설정 (~/.zprofile 또는 ~/.zshrc에 추가)
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

#### 1-2. Git 설치

```bash
# Git 설치
brew install git
```

**예상 출력**:

## 4. Git 기본 설정

### 🎯 Step 5: 사용자 정보 설정

> **실습**: Git 사용을 위한 자신의 신원을 설정하세요.

Git을 사용하기 전에 자신의 신원을 설정해야 합니다. 이 정보는 모든 커밋에 기록됩니다.

#### macOS (터미널)

```bash
# 사용자 이름 설정
git config --global user.name "Your Name"

# 이메일 설정 (GitHub 이메일과 일치하도록)
git config --global user.email "your.email@example.com"
```

#### Windows (PowerShell 또는 Git Bash)

```powershell
# PowerShell 또는 Git Bash에서
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

> **중요**: GitHub에서 이메일을 공개적으로 표시하지 않으려면 `noreply` 이메일을 사용하세요.
> ```
> git config --global user.email "username@users.noreply.github.com"
> ```

### 🎯 Step 6: 기본 브랜치 이름 설정

최근 Git은 기본 브랜치명을 `main`으로 사용합니다.

```bash
# 기본 브랜치를 main으로 설정
git config --global init.defaultBranch main
```

### 🎯 Step 7: 설정 확인

#### macOS / Windows

```bash
# 전체 설정 확인
git config --global --list
```

**예상 출력**:
```
user.name=Your Name
user.email=your.email@example.com
init.defaultbranch=main
```

### 🎯 Step 8: 줄 바꿈 설정 (선택사항)

운영체제 간의 호환성을 위해 줄 바꿈 방식을 설정합니다.

#### macOS

```bash
# 커밋 시 줄 바꿈 변환, 체크아웃 시 변환 없음
git config --global core.autocrlf input
```

#### Windows

```bash
# 커밋 시 CRLF를 LF로 변환, 체크아웃 시 LF를 CRLF로 변환
git config --global core.autocrlf true
```

> **참고**: Windows에서 Git 설치 시 "Checkout Windows-style, commit Unix-style"을 선택했다면 이미 설정되어 있을 수 있습니다.

## 6. Git 워크플로우

### 6.1 기본 워크플로우

```
1. 작업 디렉토리 (Working Directory)
   ↓ 파일 수정
2. 스테이징 영역 (Staging Area)
   ↓ git add
3. 로컬 레포지토리 (Local Repository)
   ↓ git commit
4. 원격 레포지토리 (Remote Repository)
   ↓ git push
```

### 6.2 일반적인 작업 순서

#### macOS / Windows

```bash
# 1. 최신 코드 가져오기
git pull origin main

# 2. 기능 브랜치 생성
git checkout -b feature/my-feature

# 3. 작업 및 커밋
git add .
git commit -m "Add new feature"

# 4. main 브랜치로 전환
git checkout main

# 5. 병합
git merge feature/my-feature

# 6. 원격에 푸시 (권한이 있는 경우)
git push origin main
```

## 7. 문제 해결

### 문제 1: commit 메시지 수정 필요

**증상**: 커밋 메시지에 오타가 있거나 수정이 필요함

**해결 방법**:
```bash
# 가장 최근 커밋 메시지 수정
git commit --amend -m "올바른 커밋 메시지"
```

### 문제 2: 파일을 스테이징하지 않고 커밋

**증상**: `git add`를 깜빡함

**해결 방법**:
```bash
# 수정된 파일을 스테이징과 커밋을 동시에
git commit -am "커밋 메시지"
```

### 문제 3: 잘못된 브랜치에서 작업

**증상**: main 브랜치에서 직접 작업함

**해결 방법**:
```bash
# 1. 변경사항 저장
git stash

# 2. 올바른 브랜치로 전환
git checkout feature-branch

# 3. 저장한 변경사항 복원
git stash pop
```

### 문제 4: GitHub 인증 오류 (macOS)

**증상**: 푸시 시 인증 실패

**해결 방법 - SSH 키 설정**:

```bash
# SSH 키 생성
ssh-keygen -t ed25519 -C "your.email@example.com"

# SSH 에이전트 시작 및 키 추가
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# 공개키 복사
cat ~/.ssh/id_ed25519.pub
```

1. 복사한 공개키를 GitHub에 등록:
   - GitHub → Settings → SSH and GPG keys → New SSH key
   - 키 붙여넣기

2. HTTPS 대신 SSH URL 사용:
```bash
# 기존 원격 URL을 SSH로 변경
git remote set-url origin git@github.com:sysmoon80/lec-2601-pythonai.git
```

### 문제 5: GitHub 인증 오류 (Windows)

**증상**: 푸시 시 인증 실패

**해결 방법 - Git Credential Manager**:

Windows용 Git 설치 시 Git Credential Manager가 포함되어 있습니다.

```bash
# Git Credential Manager 확인
git config --global credential.helper
```

**Personal Access Token 사용**:

1. GitHub에서 토큰 생성:
   - GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
   - Generate new token → `repo` 권한 체크 → Generate

2. 첫 푸시 시:
```bash
# 토큰으로 로그인
git push
Username: your-github-username
Password: ghp_YourPersonalAccessTokenHere
```

### 문제 6: macOS에서 Git 명령어를 찾을 수 없음

**증상**: `command not found: git`

**해결 방법**:

```bash
# Homebrew로 설치한 Git 경로 확인
which git

# 시스템 Git 대신 Homebrew Git 사용하도록 PATH 설정
# ~/.zshrc 또는 ~/.bash_profile에 추가
echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 문제 7: Windows에서 한글 파일명 깨짐

**증상**: 한글 파일명이 깨져서 표시됨

**해결 방법**:

```bash
# Git Bash에서 한글 파일명 처리 설정
git config --global core.quotepath false
```
